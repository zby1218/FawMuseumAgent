汽车博物馆讲解机器人 Agent 系统架构文档 文档版本 v1.0 日期 2026-04 状态 草稿 · 内部评审 适用场景 汽车博物馆讲解机器人 1. 系统概述 本文档描述汽车博物馆讲解机器人的 Agent 系统完整技术架构，覆盖多模态输入处理、意图识别、LangGraph 编排、LLM 推理、Skill 调用、动作执行以及记忆系统的设计方案。 1.1 核心能力 多模态输入：语音（ASR）+ 图像（VLM）联合理解 意图识别：三级级联路由，动作 < 1ms 响应 多轮对话：跨轮上下文保持，展品信息动态注入 多意图并行：一句话多指令，依赖分析后并行/串行执行 Skill 体系：结构化工具调用（播放视频、导航、展示参数等） 记忆系统：三层记忆（工作 / 会话 / 长期），借鉴 Hermes Agent 架构 1.2 技术选型总览 模块 选型 Agent 编排框架 LangGraph v1.x（锁版本） 多模态 LLM 支持 vision 的模型（如 Claude / GPT-4o / Qwen-VL） 意图分类输出 Pydantic structured output 语义路由 Semantic Router（aurelio-ai） 多轮记忆持久化 LangGraph MemorySaver + Mem0 长期记忆 Mem0（向量检索 + 事实存储） Skill 沉淀 agentskills.io 标准（Hermes 兼容） 异步执行 Python asyncio 可观测性 LangSmith 或 Phoenix（OpenTelemetry） 2. 整体架构分层 系统分为六个主要层次，自上而下依次为： 层 名称 职责 L1 多模态输入层 ASR 语音转文字；摄像头图像 / 用户上传图像；VLM 前置融合，图文统一为文本描述 L2 意图识别层 三级级联路由（关键词 → Semantic Router → LLM 兜底）；多意图分割；依赖分析 L3 LangGraph 编排层 State Machine 管理全局状态；条件边路由到各功能节点；MemorySaver 持久化 L4 LLM + Skill 执行层 LLM 处理讲解与多轮对话；Function Calling 触发 Skill；VLM 处理图像识别请求 L5 并行执行层 asyncio.gather 并行执行无依赖意图；串行执行有依赖意图；流式边解析边执行 L6 输出层 TTS 语音合成；机器人动作 SDK 调用（低延迟直达，不过 LLM） 3. L1 — 多模态输入层 3.1 输入类型 输入类型 来源 / 触发场景 语音（Audio） 访客对话，经 ASR 转文字 图像 — 摄像头帧 机器人视觉，识别访客手势 / 指向 / 展品 图像 — 用户上传 访客拍展品照片询问 图像 — QR 码 扫码触发展品详情页 纯文字（Fallback） 终端调试或文字输入模式 3.2 ASR 容错策略 语音识别误识别会影响后续意图判断，需在 ASR 输出后进行前置容错： 同音 / 近音词扩展：动作词表包含常见 ASR 误识别变体（如 转伸 → 转身） 动作类意图使用 fuzzy match，而非精确字符串匹配 多轮历史存储原始识别文字，不存 nbest 列表，避免歧义传递 3.3 图文融合处理 当输入同时包含图像和文字时，使用 VLM（支持视觉的多模态模型）进行前置融合，输出统一的文本描述，后续意图识别层统一处理文本： class MultimodalInput(BaseModel): text: str | None # ASR 文字 image: bytes | None # 图像 base64 image_source: Literal[ # 图像来源 "camera", "user_upload", "qr"] | None def input_processor_node(state): inp = state['raw_input'] if inp.image: # 图文送 VLM，输出统一文本描述 description = vlm.invoke([HumanMessage(content=[ {'type': 'image_url', 'image_url': {'url': f'data:image/jpeg;base64,...'}}, {'type': 'text', 'text': inp.text or '这是什么？'} ])]) return {'processed_text': description.content, 'has_image': True, 'last_image_context': description.content} return {'processed_text': inp.text, 'has_image': False} 关键设计：last_image_context 存入 AgentState，供后续多轮对话引用。 访客问【刚才那辆车发动机多大】时，即便无新图像，State 中仍保有图像识别结果。 4. L2 — 意图识别层 4.1 意图分类体系 Intent 类型 触发示例 处理路径 action 点头 / 转身 / 过来 关键词拦截 → action_executor exhibit_qa 这辆车是什么年代的？ LLM Agent → TTS chitchat 你叫什么名字？ LLM Agent → TTS skill 播放这辆车的视频 LLM Function Call → skill_dispatcher image_identify 这是什么车？（附图） VLM → exhibit 识别 → llm_agent gesture_action 摄像头检测到指向动作 VLM → action_executor qr_scan 扫码请求 QR 解析 → skill_dispatcher 4.2 三级级联路由 按速度和成本从低到高依次尝试，前一级命中即返回，不进入下一级： 第一级：关键词快速拦截（< 1ms） 适用于动作指令，词汇集合有限且边界清晰： ACTION_PATTERNS = { "nod": ["点头", "点点头"], "shake": ["摇头"], "turn": ["转身", "转过去", "转过来"], "come_here": ["过来", "来这里", "走过来"], "stop": ["停", "停下", "别动"], "wave": ["挥手"], } def fast_action_match(text: str) -> str | None: for action, keywords in ACTION_PATTERNS.items(): if any(kw in text for kw in keywords): return action return None # 未命中，进入第二级 第二级：Semantic Router（embedding 相似度，毫秒级） 对非动作类 intent 做粗粒度分类，置信度高于阈值（0.75）直接路由： from semantic_router import Route, RouteLayer from semantic_router.encoders import OpenAIEncoder routes = [ Route(name="exhibit_qa", utterances=["这辆车是什么年代的", "介绍一下发动机", "这个技术叫什么"]), Route(name="skill", utterances=["播放视频", "导航到下一个展区", "显示技术参数"]), Route(name="chitchat", utterances=["你叫什么名字", "你好", "今天天气怎么样"]), ] rl = RouteLayer(encoder=OpenAIEncoder(), routes=routes) result = rl(text) if result.name and result.similarity_score > 0.75: return result.name # 命中，直接路由 # 未命中，进入第三级 第三级：LLM 兜底分类（百毫秒级） 低置信度或语义复杂时，LLM 输出结构化 intent 对象： class SingleIntent(BaseModel): type: Literal['action','exhibit_qa','chitchat','skill', 'image_identify','gesture_action','qr_scan'] confidence: float action_name: str | None = None params: dict = {} depends_on: int | None = None # 依赖第几个意图 class MultiIntent(BaseModel): intents: list[SingleIntent] # 有序列表，保留执行顺序 result = llm.with_structured_output(MultiIntent).invoke([ SystemMessage(MULTI_INTENT_PROMPT), HumanMessage(processed_text) ]) 4.3 多意图检测与分割 当输入包含连接词（然后 / 顺便 / 再 / 另外 / 接着）时，触发多意图解析，LLM 一次输出完整有序意图列表，同时分析依赖关系： 示例："你走过来，然后介绍一下发动机，顺便播放视频" → intents[0]: action(come_here), depends_on=null → 立即执行 → intents[1]: exhibit_qa(发动机), depends_on=0 → 等机器人到位后执行 → intents[2]: skill(play_video), depends_on=null → 并行加载 5. L3 — LangGraph 编排层 5.1 AgentState 定义 class AgentState(TypedDict): # 核心对话 messages: Annotated[list, add_messages] intent_queue: list[SingleIntent] # 待执行意图队列 execution_results: list[dict] # 各意图执行结果 # 展品上下文 current_exhibit: str # 当前展品 ID exhibit_info: dict # 展品详细信息 # 多模态上下文 has_image: bool last_image_context: str | None # VLM 图像描述（跨轮保留） identified_exhibit: str | None # 图像识别出的展品 ID # 执行状态 pending_action: str | None # 待执行机器人动作 active_skill: str | None # 当前 skill 名称 # 记忆 visitor_profile: dict # 长期记忆：访客偏好 session_summary: str | None # 会话记忆：本次参观摘要 5.2 Graph 节点与路由 节点 职责 input_processor 多模态输入融合，VLM 图像处理 intent_router 三级级联路由，输出 intent_queue image_identifier 展品图像识别，更新 identified_exhibit action_executor 动作指令执行，调用机器人 SDK llm_agent 展品讲解、多轮对话，含 Function Calling skill_dispatcher Skill 调用分发（播视频/导航/参数展示） memory_writer 写入会话记忆和长期记忆（异步，不阻塞响应） 5.3 条件路由规则 def route_by_intent(state: AgentState) -> str: queue = state['intent_queue'] if not queue: return END intent = queue[0] # 取队首 # 置信度兜底：低于阈值统一走 LLM if intent.confidence < 0.75: return 'llm_agent' routing = { 'action': 'action_executor', 'gesture_action': 'action_executor', 'exhibit_qa': 'llm_agent', 'chitchat': 'llm_agent', 'skill': 'skill_dispatcher', 'image_identify': 'image_identifier', 'qr_scan': 'skill_dispatcher', } return routing.get(intent.type, 'llm_agent') 5.4 多轮对话持久化 使用 LangGraph MemorySaver 做 checkpoint，每次状态转换持久化，支持断线重连后恢复上下文。LLM 调用时注入动态 system prompt： def build_system_prompt(state: AgentState) -> str: exhibit = state.get('exhibit_info', {}) image_ctx = state.get('last_image_context', '') visitor = state.get('visitor_profile', {}) return f'''你是{exhibit.get('museum', '汽车博物馆')}的讲解机器人。 当前展品：{exhibit.get('name', '未知展品')} 展品信息：{exhibit.get('description', '')} 图像上下文：{image_ctx} 访客偏好：{visitor.get('interests', '未知')} 规则： - 回答简洁，适合语音播报（< 100 字为佳） - 涉及技术细节时可适当展开 - 不确定时主动表明，不编造信息 ''' 6. L4 — LLM + Skill 执行层 6.1 Skill 定义 所有 Skill 以 Function Calling 格式定义，LLM 自主决定何时调用、参数如何填写： SKILLS = [ { "name": "play_video", "description": "播放展品相关介绍视频", "parameters": {"exhibit_id": "string", "clip": "string | null"} }, { "name": "navigate_to", "description": "导航机器人到指定展区", "parameters": {"area": "string", "exhibit_id": "string | null"} }, { "name": "show_specs", "description": "在屏幕上显示车辆技术参数表", "parameters": {"exhibit_id": "string", "spec_type": "string | null"} }, { "name": "highlight_part", "description": "用激光/指示灯高亮展品某个部件", "parameters": {"part_name": "string"} }, ] 6.2 Skill 沉淀机制（借鉴 Hermes） 当 llm_agent 成功完成一次复杂讲解请求后，post-execution 评估器将优质对话路径写入 Skill 库，下次类似请求直接复用，无需重新推理： Skill 沉淀触发条件： 1. 访客正向反馈（点头 / 语音确认） 2. 完整回答了 3 步以上的多轮追问 3. 本次对话 token 消耗超过阈值（代表是复杂推理） 存储格式：agentskills.io 标准 JSON，可与 Hermes Agent 互通 7. L5 — 并行执行层 7.1 依赖分析与执行策略 async def execute_intent_queue(queue: list[SingleIntent]): independent = [i for i in queue if i.depends_on is None] dependent = [i for i in queue if i.depends_on is not None] # 无依赖意图并行执行 await asyncio.gather(*[dispatch(intent) for intent in independent]) # 有依赖意图串行执行（按 depends_on 顺序） for intent in sorted(dependent, key=lambda x: x.depends_on): await dispatch(intent) 7.2 执行策略对照 场景 策略 示例 动作 + 播放视频 并行 "过来" + "播视频" 同时开始 动作 → 讲解 串行（depends_on） "到位后" 再开口讲解 多个讲解问题 串行（有序队列） 依次回答，避免语音重叠 视频加载 + 讲解 并行 视频后台加载，讲解同步开始 7.3 流式边解析边执行 LLM 输出多意图列表时，采用流式处理模式：解析到第一个无依赖意图即立即派发，不等待完整输出。这是车机系统【流畅感】的核心技术之一。 async def stream_and_execute(user_input: str): async for partial in llm.astream_structured(MultiIntent, user_input): if partial.intents: new_intent = partial.intents[-1] # 无依赖意图立即派发，不等后续意图解析完 if new_intent.depends_on is None: asyncio.create_task(dispatch(new_intent)) 8. 记忆系统（三层架构） 借鉴 Hermes Agent 的三层记忆设计，适配博物馆讲解场景： 层级 名称 内容 存储 / 生命周期 Layer 1 工作记忆 当前对话消息、当前展品、图像上下文 LangGraph State，会话内 Layer 2 会话记忆 本次参观路径、已讲解展品列表、访客问题偏好 MemorySaver，参观结束后归档 Layer 3 长期记忆 回头客偏好、历史参观记录、兴趣标签 Mem0 向量存储，跨次参观保留 8.1 Mem0 集成 from mem0 import MemoryClient mem = MemoryClient(api_key=MEM0_KEY) # 参观结束后异步写入长期记忆（不阻塞当次响应） async def write_long_term_memory(visitor_id: str, session: dict): await mem.add( messages=session['conversation_history'], user_id=visitor_id, metadata={'exhibit_ids': session['visited_exhibits']} ) # 参观开始时加载访客偏好 async def load_visitor_profile(visitor_id: str) -> dict: memories = await mem.search( query='visitor interests and preferences', user_id=visitor_id, limit=5 ) return {'interests': [m['memory'] for m in memories]} 9. 部署与可观测性 9.1 延迟目标 路径 目标延迟 动作指令（关键词命中） < 50ms（含机器人 SDK 调用） Semantic Router 路由 < 100ms LLM 意图分类（兜底） < 500ms LLM 讲解首 token < 1s（流式输出） 多意图首个动作派发 < 200ms（流式边解析边执行） 9.2 可观测性配置 推荐使用 LangSmith 或 Phoenix（OpenTelemetry 兼容）对每个 LangGraph 节点的输入输出进行 tracing： 每次 intent_router 调用记录分类置信度和路由决策 llm_agent 节点记录 prompt token 消耗和 Skill 触发情况 action_executor 记录动作类型和执行延迟 Skill 沉淀事件单独打标，便于分析哪类讲解路径被复用 9.3 版本管理建议 重要：LangGraph 迭代频繁，务必在 requirements.txt 中锁定精确版本。 langgraph==1.x.x langchain-core==0.x.x semantic-router==0.x.x mem0ai==0.x.x 每次升级前在 staging 环境完整跑一遍意图识别覆盖率测试， 重点关注 Pydantic structured output 和 MemorySaver API 变化。 10. 后续规划 10.1 近期（1-2个月） 完成 LangGraph State Machine 核心骨架搭建 实现三级意图路由，覆盖博物馆所有 intent 类型 接入第一批 Skill（play_video / show_specs） 接入 ASR 和 TTS，完成端到端语音链路联调 10.2 中期（3-4个月） 图像输入接入，VLM 展品识别上线 多意图并行执行，流式边解析边执行 Mem0 长期记忆集成，回头客偏好识别 LangSmith / Phoenix 可观测性上线 10.3 长期（5个月+） Skill 自动沉淀，agentskills.io 格式持久化 访客反馈闭环，优质讲解路径自动识别 手势识别（摄像头 → VLM → 动作意图） 多机器人协同，A2A 协议接入（参考 Google ADK） — 文档结束 —
